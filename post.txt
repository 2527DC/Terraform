Terraform variables play a major role in creating reusable and scalable infrastructure. Instead of writing different 
code for each environment, we can simply pass values dynamically using variables â€” keeping the project clean and consistent.


In Terraform, we usually declare variables inside a file like "variables.tf", where we define:
********************************************************************************************

* Variable name
* Type
* Description
* Default value

This allows us to use one project for different environments (dev, test, prod) without modifying the main code.
*******************************************************************************************************

Terraform allows us to override default values using .tfvars files such as: (Using .tfvars Files)
****************************************************************************************************

* dev.tfvars
* prod.tfvars
* staging.tfvars

This helps maintain a single codebase while customizing values for each environment.
*************************************************************************************


Important Note When Using Modules
*********************************

A very common mistake:  Variables declared inside a module are only available inside that module. 
They are NOT automatically available in the root module.So when variables are declared inside 
"modules/vpc/variables.tf", Terraform will not apply values from the root .tfvars file unless the 
root module also declares and passes them.

Correct Approach:

* Declare variables inside the module
* Declare the same variables in the root module
* Pass values from the root module into the module
